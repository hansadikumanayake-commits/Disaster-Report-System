# Disaster Report System

## Project Description

The Disaster Report System is a web-based application developed using PHP and MySQL. The main purpose of this system is to allow users to report disaster incidents by submitting their personal details, selecting a disaster type, entering location details,uploading disaster-related photos and saving the report details in a database

The system is planned to include a login system with username, password, and an OTP generated by the system itself. This helps to improve the security of the system and allows only verified users to access the report submission form or admin panel.

The system is also planned to support GeoTag information inside uploaded photos. The GeoTag should contain the latitude and longitude of the place where the disaster incident happened.

This system is designed for disaster situations that may occur in Sri Lanka
## Main Features

* Submit disaster reports through a client form
* Login in to the system using username and password
* Generate OTP automatically by the system for verification
* Enter user details like name and contact number
* Select disaster type from a dropdown list
* Enter the location under using the district and its Grama-Niladhari    division
* Add geotag details to uploaded photos using the longitude and latitude
* Upload up to three disaster photos
* Read GeoTag metadata from uploaded photos
* Store report details in a MySQL database
* Store uploaded photos in an `uploads` folder
* Store uploaded photo paths in the database
* View all submitted reports in an admin panel
* Display uploaded photos in the admin panel
* Show the incident location on a Sri Lankan map using the saved longitude and latitude

## How the System Works

The system starts with the login page. The user enters the username and password. After that, the system generates an OTP automatically for verification. Once the OTP is verified successfully, the user can access the disaster report form.

In the disaster report form,the user enters their name and contact number, selects the disaster type from the dropdown list, enters the district and Grama-Niladhari division, and uploads up to three disaster-related photos. After clicking the submit button, the data is sent to the PHP backend. PHP saves the uploaded photos inside the `uploads` folder and stores the report details together with the photo paths in MySQL database.

The system will also save the location details of the incident using longitude and latitude. These longitude and latitude values can be used to identify the exact place where the disaster incident happened.

The admin panel retrieves all submitted disaster reports from the database and displays them as a list. This list shows the user details, disaster type,Grama-Niladhari division with the district, submitted date, and uploaded photos for each report. The admin can browse the list to review all submitted disaster reports.

## Photo GeoTag Location

In this system, uploaded disaster photos should include GeoTag information. The GeoTag means the location metadata stored inside the photo.

The GeoTag should contain the latitude and longitude of the place where the disaster incident happened. These coordinates help to identify the exact location of the incident.

When the photo is uploaded, the system should read the GeoTag details from the photo and save the latitude and longitude in the MySQL database. These saved coordinates can then be used in the admin panel to show the incident location on a Sri Lankan map or world map.

In the admin panel, each disaster report can display the uploaded photos together with their latitude and longitude. The map will use these coordinates to place a marker on the location where the incident happened.

## Technologies Used

This project uses HTML, CSS, PHP, MySQL, XAMPP

HTML is used to create the page structure, CSS is used for styling and desgin of the web pages

 PHP is used for backend processing,form submission,login validation,OTP generation,photo upload handling,GeoTag reading and database connection
 
 MySQL is used to store disaster report details like the name,telephone number,disaster type, district,Grama-Niladhari division,latitude,longitude,usernmae,password,OTP details and submitted date and time. Uploaded photos are saved in the `uploads` folder on the server, while their file paths are stored in the MySQL database 
 
 XAMPP is used to run the project locally

## Database Description

The system uses a MySQL database named `disaster_report`

Inside this database, there is a table named `disaster_reports`. This table stores the report ID, user's name, contact number, disaster type, district and Grama-Niladhari division, uploaded photo paths,latitude,longitude and submission date and time. The actual photos are saved in the `uploads` folder, while only the photo paths are stored in the database

The uploaded photos contain GeoTag metadata of longitude and latitudes.These longitudes and latitude values are then used to display disaster location on map in admin panel
